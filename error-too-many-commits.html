too many commits

version v3.4.1

This error occurs when a push directly to a branch bypassing review contains more commits than the server is able to validate in a single batch.

The recommended way to avoid this message is to use the skip-validation push option. Depending on the number of commits, it may also be feasible to split the push into smaller batches.

The actual limit is controlled by a server config option.
